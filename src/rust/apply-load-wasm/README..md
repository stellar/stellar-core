This directory contains additional Wasm contracts built for apply load benchmarking. Moving the Wasms here as opposed to Soroban test Wasms in Soroban env reduces coupling between Soroban env and Core and allows for simpler and faster iteration on the Wasms without needing to update Soroban env.

Contents:

- `token.wasm` - a token contract generated by OpenZeppeling contract wizard. Source can be found in `scripts/apply_load/token` directory.
